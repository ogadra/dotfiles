# ai-antipattern観点

AIが生成した「もっともらしいが間違っている」「過剰実装」「見かけ上の修正」を検出する。AI特有の癖に絞る。

## 対象外

以下は指摘しない。

- 局所的な冗長分岐。if/else差分呼び分け、コールバックと外部変数、1箇所用設定とループ。
- 到達不能コードや「念のため」の防御コード。
- 新パラメータの配線忘れ、依存方向、責務境界などの契約整合性。
- Fail Fast欠如。silent fallback、多段 `??`、空catch。
- 経緯コメントや説明コメントなど自然言語。
- テストが0件など、テストの不足そのもの。

## REJECTパターン

### 幻覚 / 誤API

| # | パターン | 例 |
|---|---------|-----|
| AI1 | 存在しないAPI・メソッド呼び出し | ライブラリバージョンにないメソッド |
| AI2 | 存在しない設定キー・フィールド | 参照先のスキーマ・型に定義なし |
| AI3 | 古いパターン | 非推奨API・古いバージョンのシグネチャ |
| AI4 | 生成物と手書きの混在 | Orvalとaxios直接使用が同居、生成型と手書き型が両方存在 |

### 過剰・不要

| # | パターン | 例 |
|---|---------|-----|
| AI5 | 要求されていない機能追加 | |
| AI7 | 過剰設定 | 設定可能にする必要のないものを設定可能に |
| AI8 | 不要なLegacy対応 | 明示要求なしに `LEGACY_*_MAP` 追加 |
| AI9 | ゴールドプレーティング | 求められていない改善 |
| AI10 | 早すぎるキャッシュ導入 | 計測なしでstale-while-revalidate追加 |
| AI11 | 早すぎるメモ化 | ボトルネック未特定のまま `memo` 多用 |

### 要求外の変更

| # | パターン | 例 |
|---|---------|-----|
| AI12 | 「ついで」の改名・移動・型名変更 | 変更ファイル内という理由だけの整理 |
| AI13 | UI文言・a11y名・イベント名・エラー文言の要求外変更 | |
| AI14 | フレームワーク作法改善の混入 | 属性名の一括変更がPR目的と無関係 |
| AI15 | 既存契約変更に理由・影響範囲・移行方針の記載なし | Props型名・イベント名の変更 |
| AI16 | 既存テストを新契約へ追随させるだけの変更 | 期待値の緩和 |
| AI17 | 既存コメントを「説明コメント」と決めつけて削除 | Whyを持つコメントの喪失 |

### 見かけ上の修正

| # | パターン | 例 |
|---|---------|-----|
| AI18 | 指摘の対象を修正せず、テストで指摘内容を検証するだけ | 「コメント不要」→ コメント有無を検証するテスト追加 |
| AI19 | 指摘を修正せず、ドキュメントで「意図的」と説明 | |
| AI20 | 指摘と無関係なリファクタで対応したように見せる | |
| AI21 | 仕様変更で旧仕様の不在確認テストだけ追加 | 新仕様の肯定テストがない |
| AI22 | テストダブルを緩めて成功させる | 制約が伝播していなくても通る |

### 一貫性

| # | パターン | 例 |
|---|---------|-----|
| AI23 | コードベース他所にないパターンの独自導入 | |
| AI24 | 命名・ログ・エラー・テストが既存スタイルと不一致 | |
| AI25 | 新エクスポートがindex / barrelに登録されていない | プロジェクト規約に登録手順がある場合 |

## 検出手順

1. 追加されたAPI呼び出し・import・型・設定キーが、対象のライブラリ・スキーマ・生成物に実在するか確認する。存在しなければ幻覚。
2. 変更ファイル内の改名・移動・削除が、PRタイトル・本文の要求と因果関係を持つか確認する。持たなければAI12/AI13/AI14。
3. 指摘への対応と主張するdiffが、指摘対象ファイル・対象行を実際に変更しているか確認する。テスト追加・ドキュメント追加のみで済ませていないか確認する。
4. コードベース既存の同種処理をgrepし、独自パターンが導入されていないか確認する。
5. 生成物 (Orvalなど) と手書きコードの混在がないか確認する。

## 出力ルール

- 幻覚APIを指摘する場合、確認手段 (対象パッケージのバージョン・型定義ファイル) を明示する。
- 過剰・不要の指摘は「これがなくても要求は満たせる」根拠を1文で示す。
